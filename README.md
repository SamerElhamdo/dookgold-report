
#Dook Gold

## 1. ملخص تنفيذي

آخر 50 كوميت تغطي فترة من 5 مارس إلى 18 مارس 2026، وتشمل تحسينات في الأمان، الواجهة، الترجمة، التبادل عبر الشبكات، وإدارة العملات.

---

## 2. أهم الإضافات حسب الفئات

### 2.1 الأمان والمصادقة

| الميزة | الوصف |
|--------|--------|
| **المصادقة البيومترية** | فتح المحفظة بالبصمة أو التعرف على الوجه، مع الاحتفاظ بالبيانات عند التعطيل |
| **تصدير المفتاح الخاص** | طلب كلمة المرور، أزرار إظهار/إخفاء ونسخ، وتصميم موحد مع الثيم |
| **قفل تلقائي قابل للتعديل** | خيارات: دقيقة، 5، 15، 30 دقيقة، ساعة، أو أبداً |
| **إعادة تصميم بطاقات الأمان** | بطاقات تصدير المفتاح وحذف المحفظة بتصميم أوضح |
| **أيقونة حذف المحفظة** | استخدام أيقونة Trash2 بدل النص |

### 2.2 الترجمة والواجهة (i18n)

| الميزة | الوصف |
|--------|--------|
| **ترجمة شاملة** | ترجمة النصوص في شاشات البصمة، برنامج الشركاء، إدارة العملات |
| **رسائل المعاملات** | ترجمة رسائل التبادل والجسر |
| **استبدال الإيموجي** | استبدال الإيموجي بأيقونات (مثل شاشة السجل) |
| **دعم لغات متعددة** | عربي، إنجليزي، فرنسي، ألماني، صيني، إسباني، ياباني، هندي، روسي، أردو |

### 2.3 التصميم والثيم

| الميزة | الوصف |
|--------|--------|
| **توحيد Border Radius** | استخدام `rounded-xl` للأزرار والحقول |
| **ألوان الثيم الداكن** | نص أبيض على الأزرار الأساسية في الوضع الداكن |
| **تحسين السويتش** | دائرة بيضاء، إطار بلون البرايم عند التعطيل |
| **تصميم موحد** | تطبيق ألوان الثيم على بطاقات وأزرار متعددة |

### 2.4 التبادل والمحفظة

| الميزة | الوصف |
|--------|--------|
| **1inch Fusion+ Cross-Chain** | مبادلة عبر الشبكات مع واجهة عرض وتتبع الطلبات |
| **تنفيذ المبادلة على السلسلة** | تنفيذ عمليات التبادل مباشرة على الشبكة |
| **دعم عنوان العقد** | إمكانية إضافة عملات عبر عنوان العقد ERC-20 |
| **تجمع المحفظة متعدد الشبكات** | عرض الأرصدة عبر ETH, BSC, Polygon, Arbitrum, Optimism, Base |
| **تصحيح Chain IDs** | تصحيح معرفات Arbitrum, Optimism, Base |
| **انتهاء صلاحية الطلبات** | انتهاء صلاحية الطلبات غير المنشورة بشكل صحيح |

### 2.5 إدارة العملات والشاشات

| الميزة | الوصف |
|--------|--------|
| **شاشة إدارة العملات** | تحميل العملات من API المحفظة، تفعيل/تعطيل بدون حذف البيانات |
| **CoinGecko API** | دمج API للعملات الشائعة وبيانات العقود |
| **بطاقات العملات** | عرض كل شبكة كبطاقة منفصلة مع شارة الشبكة |
| **إعادة تصميم شاشة الإرسال** | واجهة مشابهة لشاشة التبادل مع منتقي العملات |
| **فلترة المعاملات الصغيرة** | فلترة المعاملات الصغيرة في شاشة السجل |

### 2.6 السجل والمعاملات

| الميزة | الوصف |
|--------|--------|
| **سجل متعدد الشبكات** | دعم ETH, BSC, Polygon, Arbitrum, Optimism, Base, Solana |
| **تحسين دقة العرض** | تحسين عرض القيم والمعاملات في عدة شاشات |
| **تكامل Etherscan V2** | تحديث تكوين الشبكات لتكامل Etherscan V2 |

### 2.7 البنية التحتية والتطوير

| الميزة | الوصف |
|--------|--------|
| **سكربت التطوير** | إصلاح `npm run dev` في الـ backend (استخدام `dev:watch` بدل `docker compose` داخل الحاوية) |
| **Docker Hot-Reload** | وضع تطوير مع إعادة تحميل تلقائية |

| **تقييد المنافذ** | تحديث docker-compose لتقييد المنافذ المعرّضة |

## 3. خطة اختبار خطوة بخطوة

### المرحلة 1: البداية — إنشاء المحفظة

1. **الشاشة الرئيسية** (`/`)
   - التحقق من شاشة البداية والانتقال إلى الترحيب.

2. **الترحيب** (`/welcome`)
   - اختيار "إنشاء محفظة" أو "استيراد محفظة".

3. **إنشاء المحفظة** (`/create`)
   - إنشاء محفظة جديدة.
   - تسجيل عبارة الاسترداد (12 كلمة).
   - التأكيد عبر اختيار الكلمات الصحيحة.

4. **تعيين كلمة المرور** (`/set-password`)
   - إدخال كلمة مرور قوية (8 أحرف على الأقل).
   - تأكيد كلمة المرور.

5. **فتح المحفظة**
   - إدخال كلمة المرور أو استخدام البصمة (إن كانت مفعّلة).

---

### المرحلة 2: الواجهة الرئيسية

6. **شاشة المحافظ** (`/wallets`)
   - عرض إجمالي الرصيد.
   - عرض بطاقات العملات حسب الشبكات.
   - التنقل بين الإجراءات (إرسال، استلام، تحويل، إلخ).

7. **شاشة الرصيد** (`/balance`)
   - عرض تفاصيل الرصيد.
   - التحقق من الأسعار المعروضة.

---

### المرحلة 3: الإرسال والاستلام

8. **شاشة الإرسال** (`/send` أو `/send/ETH`)
   - اختيار العملة والشبكة.
   - إدخال العنوان والمبلغ.
   - التحقق من تقدير الغاز.
   - إتمام الإرسال (أو إلغاؤه في بيئة الاختبار).

9. **شاشة الاستلام** (`/deposit`)
   - عرض عنوان المحفظة ورمز QR.
   - نسخ العنوان.

---

### المرحلة 4: التبادل

10. **شاشة التبادل** (`/exchange`)
    - اختيار الشبكة والعملة المصدر والهدف.
    - إدخال المبلغ.
    - التحقق من السعر والرسوم.
    - تأكيد المبادلة (أو إلغاؤها).

11. **شاشة تأكيد المبادلة** (`/swap/confirm`)
    - مراجعة التفاصيل قبل التنفيذ.

---

### المرحلة 5: السجل والإعدادات

12. **شاشة السجل** (`/history`)
    - اختيار الشبكة (ETH, BSC, POL, ARB, OP, BASE, SOL).
    - تطبيق الفلاتر (الكل، وارد، صادر، مبادلة).
    - اختيار الفترة الزمنية.
    - التحقق من حالة "لا توجد معاملات" مع الأيقونة الصحيحة.

13. **الإعدادات** (`/settings`)
    - التنقل بين الأقسام المختلفة.

14. **الأمان** (`/security`)
    - المصادقة البيومترية.
    - تصدير المفتاح الخاص.
    - قفل تلقائي.
    - حذف المحفظة.

15. **إدارة العملات** (`/coins/manage`)
    - تفعيل/تعطيل العملات.
    - البحث عن عملة أو عنوان عقد.
    - التحقق من أن التعطيل لا يحذف البيانات.

---

### المرحلة 6: ميزات إضافية

16. **برنامج الشركاء** (`/affiliate`)
    - عرض رمز الإحالة.
    - نسخ ومشاركة الروابط.
    - التحقق من الترجمة والألوان.

17. **دفتر العناوين** (`/address-book`)
    - إضافة جهة اتصال.
    - تعديل وحذف جهات الاتصال.

18. **اللغة والثيم**
    - تغيير اللغة (`/language`).
    - تغيير الثيم (`/theme`) والتحقق من الوضع الداكن.
